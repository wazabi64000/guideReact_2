01 — Introduction & Configuration

Objectif de cette étape (fil rouge)

Préparer l'environnement et comprendre les choix technologiques pour construire l'application Animaux (liste d'animaux, filtrage, navigation simple).

Qu'est-ce que React ? (théorie synthétique)

React est une bibliothèque JavaScript pour construire des interfaces utilisateurs déclaratives et performantes. Les points essentiels :

  • Approche par composants réutilisables.
  • Gestion locale de l'état (state) et propagation via props.
  • Virtual DOM : minimisation des opérations coûteuses sur le DOM réel.
  • Écosystème riche (router, state management, testing).

Qu'est-ce que Vite ? (théorie synthétique)

Vite est un build tool moderne optimisé pour le développement front-end :

  • Serveur de développement ultra-rapide (HMR — Hot Module Replacement).
  • Build de production optimisé (esbuild + rollup).
  • Configuration minimale pour démarrer rapidement.

Pourquoi React + Vite pour le projet ?

  • Temps de feedback réduit en dev (Vite HMR) → productivité.
  • Component-driven development → facilité d'évolution du fil rouge.
  • Interopérabilité avec bibliothèques usuelles (react-router, fetch, etc.).

Installation — créer le projet (commande)

// crée un projet Vite + React (sélectionner "react" puis "javascript" ou "typescript")
npm create vite@latest animaux-app

// se déplacer dans le dossier, installer les dépendances et démarrer le serveur
cd animaux-app
npm install
npm run dev // serveur de dev (par défaut http://localhost:5173)

Structure minimale (après create)

animaux-app/
├─ index.html
├─ package.json
├─ vite.config.js
└─ src/
   ├─ main.jsx        // point d'entrée React
   ├─ App.jsx         // composant racine
   └─ assets/

Checklist de validation

  • Le serveur `npm run dev` répond sur http://localhost:5173.
  • La page d'accueil fournie par Vite s'affiche.
